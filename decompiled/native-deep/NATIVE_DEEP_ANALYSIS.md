# التحليل العميق للكود الأصلي (Native Deep Analysis) — libengine.so

> جولة هندسة عكسية متقدمة باستخدام **Ghidra 12.1.2** (فك إلى لغة C) + تحليل ELF + استخراج شامل.
> الهدف: فكّ كل شيء بأقصى وضوح ممكن، بما في ذلك المكتبة الأصلية المحمية `libengine.so`.

---

## 1. الملخّص التنفيذي

| البند | النتيجة |
|-------|---------|
| المكتبة | `lib/arm64-v8a/libengine.so` (8.2MB) |
| المُترجِم | Android **NDK r25b** / **clang 14.0.6** (من قسم `.comment`) |
| نوع الحماية | **تعتيم على مستوى الكود المصدري (OLLVM-style)** — وليس حزمة تغليف تجارية (Packer) |
| عدد الدوال | **2418** دالة |
| تم فكّها إلى C | **2242 دالة (~93%)** — الملف: `ghidra/libengine_decompiled.c` (5MB) |
| لم تُفك | 17 دالة عملاقة (~175KB لكل منها) = دوال تهيئة/فك تشفير مُسطّحة (OLLVM) |
| حِزم مخفية (DEX/Payload) | **لا يوجد** — لا DEX ثانوي ولا حمولة مضغوطة/مشفّرة منفصلة |

**الخلاصة:** الكود موجود بالكامل داخل الملف (غير مُغلّف)، لكنه **مُعتَّم بعنف** بحيث يستحيل استرجاع مصدر C نظيف تلقائيًا. التحليل الكامل للسلوك يتطلّب **تحليلًا ديناميكيًا** (Frida/مصحّح على جهاز حقيقي).

## 2. تقنيات الحماية المكتشفة (Anti-Reversing)

تم اكتشافها من خلال فكّ الكود إلى C وتحليله:

1. **نداءات نظام مضمّنة (Inline Syscalls):** `2266` موضعًا يستخدم `svc #0` مباشرة (تظهر في كود Ghidra باسم `CallSupervisor(0)`).
   - الهدف: تجاوز مكتبة libc والتحايل على الـ Hooking (مثل Frida/Xposed) الذي يعترض دوال libc.
2. **توليد كود وقت التشغيل / كود ذاتي التعديل (Self-Modifying Code):** `456` موضعًا يكتب تعليمات ARM64 من نوع فرع (`B`) ديناميكيًا (النمط `... | 0x14000000`).
   - الهدف: إخفاء تدفّق التحكّم الحقيقي حتى لا يظهر في التحليل الساكن.
3. **تسطيح تدفّق التحكّم (Control-Flow Flattening):** دوال ضخمة (حتى 175KB للدالة الواحدة) مع جداول قفز غير قابلة للاسترجاع (`Could not recover jumptable`) ومسارات ميتة — بصمة مميّزة لـ OLLVM.
4. **تشفير السلاسل النصية (String Encryption):** أسماء دوال JNI (`ac`, `aior`, `awl`, `chl`, `djp`, `eio`, `ilil`, `pjowqpxe`, `update`) **غير موجودة كنصوص صريحة** في الملف — تُفكّ وتُبنى وقت التشغيل عبر عمليات XOR/إزاحة على ثوابت (`DAT_...`).
5. **قسم مخصّص `.mytext`:** قسم غير قياسي (244 بايت) يحوي كعب (stub) كود صغير — غالبًا مساعد للفك/التسجيل.
6. **`BIND_NOW` / Full RELRO:** ربط فوري لجدول الرموز (يصعّب اعتراض الاستدعاءات عبر GOT).

## 3. نقطة الدخول `JNI_OnLoad`

- الحجم: **12,388 بايت** (دالة واحدة عملاقة — مؤشّر تسطيح تدفّق).
- الملف المستخرج: `ghidra/JNI_OnLoad.c`.
- ما تفعله (كما ظهر من الفكّ):
  - تشغّل حلقة تُنشئ صفحات قابلة للتنفيذ وتكتب فيها تعليمات فرع ARM64 ديناميكيًا (توليد Trampolines).
  - تفكّ ثوابت مشفّرة في الذاكرة ثم تتحقّق من طول سلسلة (`strlen == 0xB` أي 11 حرفًا).
  - تُسجّل الدوال الأصلية عبر `RegisterNatives` **بأسماء مفكوكة وقت التشغيل** — لذا لا يمكن ربط كل دالة Java أصلية بعنوانها الأصلي إلا ديناميكيًا.

> **قيد مهم:** خرائط `RegisterNatives` (اسم الدالة → المؤشّر الأصلي) تُبنى وقت التشغيل، ولا يمكن استخراجها ساكنًا. الحل: Frida hook على `RegisterNatives` أثناء التشغيل.

## 4. بنية الملف (ELF Layout) وتحليل الإنتروبيا

| القسم | الحجم | الإنتروبيا | ملاحظة |
|-------|-------|-----------|--------|
| `.text` | 7.8MB | 6.91 | الكود (مُعتَّم) — يشكّل ~99% من الملف |
| `.rodata` | 208KB | 3.27 | بيانات للقراءة فقط (غير مشفّرة بالجملة) |
| `.mytext` | 244B | 5.26 | **قسم مخصّص غير قياسي** |
| `.init_array` | 352B | — | **44 مُنشئ** (تُملأ عناوينها عبر Relocations) |
| overlay | 1731B | — | بيانات ملحقة بعد آخر قسم |

- **44 مُنشئ (Constructors)** يُنفَّذون قبل `JNI_OnLoad`، متباعدة بانتظام (~0x2d000) وبأحجام متطابقة (175KB) — دوال تهيئة/فك تشفير مُولّدة آليًا بواسطة أداة التعتيم.
- **لا يوجد قسم عالي الإنتروبيا (>7.2)** يدل على حمولة مضغوطة/مشفّرة — أي لا يوجد Packer تقليدي.

## 6. نموذج عمل التطبيق (من سلاسل libapp.so — طبقة Dart)

**Snake Engine** = أداة تعديل/غش للألعاب تُباع عبر نظام **بائعين/موزّعين (Seller/Reseller)** باشتراكات مرتبطة بالجهاز:

- تسجيل دخول البائع (Seller Login) و **Seller ID** و "Global Seller".
- **أكواد تفعيل (Activation Codes)** مرتبطة بجهاز واحد (device-locked).
- **اشتراكات (Subscriptions)** بمستويات (Tiers)، تحويل الاشتراك، ربط/إزالة الأجهزة.
- نظام **حظر (BANNED)** للأجهزة، ورصيد (Balance) وشحن (Topup) عبر `snakeengine.com/topup/`.
- الواجهة الخلفية: `https://rest.snakeseller.com/api/request/`.
- دعم لغات متعدّد: الإنجليزية، الإسبانية، الفلبينية (Tagalog)، الملايوية.

## 8. توصيات للتحليل الديناميكي (لتجاوز التعتيم)

الحدّ الأقصى الممكن ساكنًا تم بلوغه. لكشف السلوك الحقيقي رغم التعتيم:

1. **Frida على جهاز/محاكي arm64:**
   - Hook على `RegisterNatives` لاستخراج خريطة (اسم الدالة الأصلي ↔ المؤشّر).
   - استخدام `dart-blutter/blutter_frida.js` (من المجلد السابق) لتتبّع دوال Dart.
2. **اعتراض السلاسل المفكوكة** بعد فكّها في الذاكرة (تجاوز تشفير السلاسل).
3. **تفريغ الذاكرة (Memory dump)** بعد تنفيذ المُنشئات لالتقاط الكود بعد فك تعديله ذاتيًا.
4. تحميل `libengine_decompiled.c` في محرّر مع الاستعانة بـ `function_inventory.tsv` لتحديد الدوال ذات الاستدعاءات الكثيرة (نقاط دخول منطقية).
