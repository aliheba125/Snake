# التحليل الساكن الأقصى لـ libengine.so

توثيق كامل لجولة تحليل ساكن مكثّفة على `libengine.so` (محرّك الحماية/التحزيم)،
باستخدام أقوى الأدوات مفتوحة المصدر المتاحة، بهدف فكّ ما عجز عنه Ghidra والوصول لأقصى
ما يمكن استرجاعه ساكنياً.

## الأدوات المستخدمة (منفّذة فعلياً)
- **capstone 5.0.7** — تحليل ARM64 وتحليل بنيوي
- **pyelftools** — تحليل ELF/المقاطع/الأقسام
- **unicorn** — محاكاة جراحية (CPU) مع تحييد مقاومة المحاكاة
- **angr 9.2.102** — استرجاع CFG + مفكّك (كسر تسطيح OLLVM)
- تحليل انتشار ثابت (constant propagation) مخصّص

## أبرز النتائج
1. **مخطّط التشفير مُسترجَع بالكامل**: `AES-256` بمفتاح `= SHA-256( PRNG_xorshift(seed1, seed2) )`.
   - AES مؤكّد (MixColumns `x<<1 ^ 0x1b` + S-box @ `DAT_009281a8`).
   - SHA-256 مؤكّد (جدول K عند إزاحة الملف `0x2b450`).
   - KDF مُسترجَع حرفياً من `FUN_00161788` (ثابت النسبة الذهبية `0x9e3779b1`).
2. **فكّ دالتين** من الـ42 التي عجز عنها Ghidra: `_INIT_16` و`_INIT_36` (بـ angr).
3. **الدوال العملاقة الـ26 نسخ متطابقة 95.6%** (روتين واحد مكرّر — تضخيم مضاد للتحليل).
4. **4 دوال `FUN_` خارج حدود الملف** — كود يُولَّد وقت التشغيل، مستحيل ساكنياً (لا بايتات).

## الحدود التقنية
- ✅ الخوارزمية والبنية والـKDF: مُسترجَعة ومؤكّدة.
- ⏳ فكّ بيانات/سلاسل فعلية ساكنياً: غير ممكن تقنيًا — البذرتان والنص المشفّر يُنتَجان وقت التشغيل
  (حقول كائن، ملفات/شبكة)، ولا يوجد زوج ثابت (بذرة+نص) في الملف.
- ▶️ مسار الفكّ الفعلي: التقاط البذور والنص عبر Frida على بيئة arm64 حقيقية.

## سير العمل للفكّ الفعلي (على جهاز/بيئة arm64 حقيقية)
انظر الدليل الكامل في `../dynamic/DYNAMIC_SETUP.md`. باختصار:
```
1) frida -U -f <package> -l ../dynamic/frida_capture.js --no-pause
   -> يلتقط seed1, seed2 والنص المشفّر، ويطبع النص المفكوك مباشرةً من out_vec
2) (اختياري) python3 crypto_scheme.py 0x<seed1> 0x<seed2> <ciphertext_hex>
   -> يشتقّ المفتاح ويفكّ (يتطلب: pip install pycryptodome)
```
